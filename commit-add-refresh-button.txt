2. Skill Scores Cleanup Migration ✅
   Problem: Existing skill_scores have malformed names
   - Database: '["Exceptional communication skills"'
   - Should be: 'Exceptional communication skills'
   - Prevents getSkillLevel() from matching
   
   Solution: SQL migration to clean up data
   - Find all skill_scores with brackets
   - Remove leading [" and trailing "]
   - Remove extra quotes
   - Update in place
   
   Migration: migrations/clean-skill-scores.sql
   ```sql
   UPDATE cv_sections
   SET content = (
     SELECT jsonb_agg(
       jsonb_build_object(
         'name', 
         regexp_replace(
           regexp_replace(
             (skill->>'name')::text,
             '^\\["|"\\]$',
             '',
             'g'
           ),
           '^"|"$',
           '',
           'g'
         ),
         'level', 
         (skill->>'level')::int
       )
     )
     FROM jsonb_array_elements(content::jsonb) AS skill
   )
   WHERE section_type = 'skill_scores'
     AND content::text LIKE '%[""%';
   ```
   
   Impact:
   ✅ Cleans up existing bad data
   ✅ Skill names match template skills
   ✅ getSkillLevel() can find matches
   ✅ Custom levels will display correctly
